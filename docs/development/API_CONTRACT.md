# API Contract (OpenAPI)

The frontend is a **typed consumer** of the FastAPI backend. The backend's
OpenAPI schema is the single source of truth for every request path, param,
body, and response; the frontend generates its types from it.

## Artifacts

| File | What it is | Generated by |
|------|------------|--------------|
| `backend/openapi.json` | The API contract (source of truth) | `backend/scripts/export_openapi.py` |
| `frontend/src/api/schema.d.ts` | TypeScript types for every operation | `npm run gen:api` (openapi-typescript) |
| `frontend/src/api/client.ts` | Thin typed `fetch` client (openapi-fetch) | hand-written, once |

Both generated files are **committed** and kept in sync by CI
(`.github/workflows/openapi-contract.yml`) — a drift between the code and either
artifact fails the build.

## When you change the backend API

Any change to routes, params, or Pydantic response models changes the contract:

```bash
# 1. Regenerate the committed spec from the app
cd backend && python scripts/export_openapi.py

# 2. Regenerate the frontend types from that spec
cd ../frontend && npm run gen:api

# 3. Commit both regenerated files with your change
```

If you forget, the `OpenAPI Contract` CI job fails with a diff showing what's
stale. `python scripts/export_openapi.py --check` reproduces the backend check
locally.

## Using the typed client in the frontend

```js
import { api } from '../api/client';

// path, query/path params, body, and response are all checked against the spec
const { data, error } = await api.GET('/matchmaking/my-matches', {
  params: { query: { status: 'pending' } },
  headers: { Authorization: `Bearer ${token}` },
});
// `data` is typed from the spec; `error` holds the parsed error body on failure
```

Auth stays per-request (a `Bearer` token from Auth0 via `acquireAccessToken`), so
the client itself is stateless. `src/hooks/useMatchmaking.js` is the reference
implementation; other call sites migrate to the typed client incrementally.

## Notes

- Operational probes (`/health`, `/healthz`, `/ready`, `/health/external`) are
  excluded from the schema (`include_in_schema=False`) — they're monitoring
  endpoints, not part of the app contract, and multi-method (`GET`+`HEAD`) routes
  otherwise produced duplicate `operationId`s that break client generators.
- The spec is served live at `/openapi.json`, with Swagger UI at `/docs` and
  ReDoc at `/redoc`.
